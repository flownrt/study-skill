How it works

Learning that compounds

Six ideas separate a tutor from a chatbot that happens to know things.

1

Lives on disk, not in a chat

Each subject gets a folder: goal, sources, lessons, progress, cheat sheets. Close the tab and nothing is lost - the next session reads it back and recognizes you.

2

Grounded, not guessed

Lessons are built from trusted sources and carry citations, instead of leaning on the model's hazy memory. What you learn is anchored to something real.

3

Prove it, then progress

A lesson is not "done" when you read it. You demonstrate the skill back in chat; only real evidence turns a concept green on your map.

4

Interactive by default

Every lesson is a hand-built page with a mechanic to work through - like the KV-cache chart above - never a wall of passive text.

5

Teaches at your edge

It reads what you have already proven and teaches the next thing just past it, with spaced review so hard-won knowledge does not quietly fade.

6

Bring your own curriculum

Ships with starter maps for LLM architecture, chess and practical AI - and any syllabus or book you supply always takes priority.

Install

Three ways in

study works with any agent that supports Agent Skills. It works best where Claude has folder access and can persist files between sessions - Claude Code or Cowork.

1 · From the marketplace (recommended)

/plugin marketplace add flownrt/skills
/plugin install study@skills

One managed install that stays updated. In Cowork, add it from Customize → Personal plugins → + → Add marketplace, then enter the repo.

2 · Download the bundle

Download study.skill and install it through the Claude app's skill installer. No marketplace needed.

3 · As a folder

Copy the skill/ directory from the repo into your Claude skills directory (rename it to study). Picked up the same session, no restart.

Recommended model: Opus with extended reasoning - the curriculum reasoning and hand-built lessons reward it.
