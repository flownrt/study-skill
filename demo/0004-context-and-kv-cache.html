LLM Architecture · Tier 1 Mechanics · Lesson 04

The model's two memories

Context window & the KV cache – the one idea that explains why putting the right thing in the prompt beats hoping the weights "know" it.

Goal: hold your own in technical LLM discussions. Here you get the mental model behind long-context cost, prompt caching, and "why can't it just remember?" – plus the single reason long context is expensive at all.

Idea Weights are what it learned. Context is what it's looking at.

A model has two very different kinds of memory, and almost every design argument hinges on telling them apart.

⬤ Weights — long-term memory

Everything absorbed in training, baked into billions of frozen numbers.

Vast, but recalled hazily and never updated at chat time.

recall: the capital of france is proba…

⬤ Context — working memory

Whatever is in the window right now: your prompt, retrieved docs, the conversation so far.

Small, but available at full fidelity — verbatim, this turn only.

read: "the order number is 4471-Z"

The KV cache is the physical machinery of that working memory. As the model reads each token it computes a key and value for it in every attention layer, and keeps them around. When it generates the next token, it attends to those stored keys and values instead of re-reading the whole context from scratch. No cache would mean re-processing the entire prompt for every single new token.

The mental modelWeights = hazy long-term memory. Context = sharp working memory. The KV cache is what makes working memory physically affordable — and, because each conversation's cache is unique, it's also the hard bottleneck of long context.

Work is in units of "token-passes" — one unit = the model looking at one token once. With the cache, generating token N costs 1 pass (attend to the N−1 cached slots); without it, token N re-reads all N tokens, so the total is N(N+1)/2 ≈ N²/2. Cache memory, by contrast, only ever grows as N. Illustrative counts, not a specific model's numbers.
So what's the catch?That green line still climbs, and the cache strip never shrinks: memory grows with every token and can't be shared between conversations (each cache is unique to its own context). That linear-but-unbounded growth — not compute — is what makes very long context expensive, and it's exactly what prompt caching (reusing a stable prefix's KV) and newer attention designs try to claw back.
